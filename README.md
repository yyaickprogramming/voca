# Voca

Веб-сервис для генерации идей стартапов и SaaS-проектов, сохранения лучших вариантов, ведения задач и отслеживания прогресса.

Текущая версия сервера рассчитана на SQLite. Это удобно для дешевого деплоя на одном приложении без отдельной managed-базы.

## Локальный запуск

```bash
npm install
npm start
```

По умолчанию сервис запускается на `http://localhost:3000`.
Переменные из локального файла `.env` подхватываются автоматически.

## Переменные окружения

Смотри [.env.example](./.env.example).

Основные:

- `PORT` — порт сервера
- `SESSION_SECRET` — секрет для сессий
- `SITE_URL` — публичный адрес сайта
- `DATABASE_PATH` — путь до файла SQLite

## Перед загрузкой в GitHub

- не загружай `.env`, он уже добавлен в `.gitignore`
- не загружай `node_modules`
- не загружай локальную SQLite-базу из папки `database`
- в репозитории должны остаться код, `package.json`, `package-lock.json`, `.env.example`

## Деплой на Amvera

Для запуска без отдельной базы:

1. В `Конфигурации` выставить:
   - `command`: `npm start`
   - `containerPort`: `3000`
   - `persistenceMount`: `/data`
2. В `Переменные` добавить:
   - `NODE_ENV=production`
   - `SESSION_SECRET=<длинная случайная строка>`
   - `SITE_URL=<публичный адрес сайта после выдачи домена>`
   - `DATABASE_PATH=/data/voca.sqlite`
3. Запустить сборку.

Файл базы будет лежать в постоянном хранилище `/data`, поэтому отдельная PostgreSQL-услуга не нужна.

## Что проверить после запуска

- открывается главная страница
- работает регистрация
- работает вход
- сохраняются идеи
- создаются задачи
- маршрут `/health` возвращает статус `ok`

## Важно для роста проекта

SQLite хорошо подходит для MVP и одного инстанса. Когда пользователей станет заметно больше, следующим шагом лучше будет миграция обратно на PostgreSQL.
